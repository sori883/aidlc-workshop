# ペルソナ (Personas)

本システムは認証・権限管理を持たないため、利用者は単一ロールとして扱う。

## Persona 1: 予約する社員 (Booking Employee)

- **役割**: 会議室を利用したい社内メンバー。同時に、会議室マスタの登録・整備も自分たちで行う（専任管理者は置かない）。
- **目的 (Goals)**:
  - 使いたい会議室を、希望の日時で確実に確保したい。
  - 空いている会議室を素早く見つけたい。
  - 不要になった予約を取り消したい。
  - 利用対象の会議室（名前・収容人数・設備・場所）を登録・整備したい。
- **困りごと (Pain Points)**:
  - 共有スプレッドシート運用でダブルブッキングが頻発し、当日に会議室が使えないことがある。
  - 誰がどの時間に予約したか分かりにくい。
  - 空き会議室を目視で探すのが手間。
- **技術レベル**: REST API（Swagger UI 含む）を操作できる程度の社内利用者。
- **利用文脈**: ローカル実行の社内ツール。認証はなく、予約者は名前/メール等の文字列で自己申告する。
