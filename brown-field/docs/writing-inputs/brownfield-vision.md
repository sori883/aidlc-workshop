# Vision: 定期予約機能の追加 — 社内会議室予約システム

## Current State（現状）
Python + FastAPI + SQLite の会議室予約 REST API。
会議室の CRUD、予約の作成・一覧・キャンセル、
半開区間によるダブルブッキング防止（重複時 409）、
空き会議室検索を提供している。フロントエンドはない。
テストは pytest で、重複判定の境界条件を中心にカバー。

## What We Are Adding（追加するもの）
毎週同じ曜日・時間帯に繰り返す定期予約を作成できる機能。
定期予約はシリーズとして管理し、シリーズ単位・個別回単位の
両方でキャンセルできる。

## In Scope（今回のスコープ）
- 週次の繰り返し予約の作成（終了日 or 回数指定）
- シリーズ内の1回だけが重複する場合の扱い（409 で全体拒否 or
  スキップ可能かは要件分析で決定）
- シリーズ全体のキャンセル / 個別回のキャンセル
- 予約一覧・詳細でのシリーズ情報の表示

## Out of Scope（スコープ外）
- 隔週・月次などの複雑な繰り返しルール
- 認証・権限、キャンセルポリシー、営業時間制約
- タイムゾーン対応

## What Must Not Change（変更禁止）
- 既存の単発予約 API のリクエスト/レスポンス契約
- ダブルブッキング防止の半開区間ロジック（隣接OK・重なりNG）
- README 記載の HTTP ステータス方針
- 既存テストはすべてパスし続けること。既存テストの改変は不可

## Open Questions（未確定事項）
- シリーズの一部が重複する場合、全体拒否かスキップ作成か
- シリーズは新テーブルか、reservations への列追加か
