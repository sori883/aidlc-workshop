# Performance Test Instructions

## 適用可否: N/A（今回のスコープ外）

本機能は社内向け・小規模・ローカル SQLite 実行であり、明示的な性能 SLA・負荷要件を持たない
（requirements NFR-5、nfr-requirements.md NFR-RS-5）。したがって正式な負荷/ストレステストは
実施しない。

## 軽量な性能上の考慮（記録のみ）
- シリーズ作成時の重複チェックは最大 52 回 × 対象会議室の active 予約走査。
  既存 `ix_reservations_room_id_status` インデックスを活用し、小規模データでは実用上問題なし。
- 上限（52回）により1リクエストの処理量が有界。

## 将来的に性能検証が必要になった場合
- ツール例: locust / k6 でシリーズ作成・空き検索の応答時間を計測。
- ボトルネック候補: `has_conflict` のアプリ側全走査（大規模データ時）。その際は SQL 側での
  重複判定（時間範囲クエリ）へ最適化を検討。
