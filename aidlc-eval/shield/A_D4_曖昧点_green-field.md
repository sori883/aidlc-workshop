# D4 曖昧点リスト — green-field（会議室予約システム構築）（シールド）

元指示（開発セッションJSONLの冒頭メッセージ。原文を末尾に同梱）の曖昧点・未確定事項を事後に定義したもの。
採点時のみ使用し、実験セッションには見せない（事後検査のため実質的な漏洩リスクはないが、
再実行時の統制のため原則は維持する）。

元指示の要点: 会議室の登録・予約・キャンセル、重複防止が必須。Python+FastAPI+SQLite、
REST APIのみ、2〜3時間で完成。「細かい仕様（定期予約、権限、キャンセルポリシーなど）は
決めていないので、質問してください」と明示。

## 曖昧点一覧

| # | 種類 | 論点 |
| --- | --- | --- |
| P1 | 明示未決 | 定期予約（繰り返し予約）を今回のスコープに含めるか |
| P2 | 明示未決 | 権限・認証。予約者をどう識別するか（認証なしで誰でも操作できてよいか） |
| P3 | 明示未決 | キャンセルポリシー。誰が・いつまでキャンセルできるか、再キャンセル時の挙動（冪等性） |
| P4 | 暗黙 | 重複判定の境界。前の予約の終了時刻と次の開始時刻が同じ（隣接）場合を重複とみなすか |
| P5 | 暗黙 | 時刻の扱い。予約時間の粒度（分単位か枠制か）、過去日時の予約を許すか |
| P6 | 暗黙 | 会議室の属性。名前以外に何を持つか（定員・設備・場所など） |
| P7 | 暗黙 | 空き検索。必須要件に含まれていないが必要か（「重複を防げること」だけでは検索の要否が決まらない） |
| P8 | 暗黙 | エラー応答仕様。重複時・検証エラー時のHTTPステータスコードとレスポンス形式 |
