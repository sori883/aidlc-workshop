# D4 曖昧点リスト — brown-field（定期予約機能の追加）（シールド）

元指示と brownfield-vision.md（docs/writing-inputs/）の曖昧点・未確定事項を事後に定義したもの。
採点時のみ使用し、実験セッションには見せない。

Visionの要点: 週次の定期予約をシリーズとして追加。終了日 or 回数指定。シリーズ/個別回の
キャンセル。既存API契約・重複判定ロジック・既存テストは変更禁止。
Vision自身が「Open Questions」として2点を明示している（P1・P2）。

## 曖昧点一覧

| # | 種類 | 論点 |
| --- | --- | --- |
| P1 | Vision明示 | シリーズの一部の回が既存予約と重複する場合、全体拒否か、重複回だけスキップして作成か |
| P2 | Vision明示 | データモデル。シリーズを新テーブルにするか、既存 reservations への列追加にするか |
| P3 | 暗黙 | 終了条件の指定方法。`count` と `until` は排他か、両方指定・両方未指定のときどうするか |
| P4 | 暗黙 | 繰り返し回数の上限。無制限を許すか、上限値はいくつか |
| P5 | 暗黙 | 過去日時の扱い。最初の回（または一部の回）が過去になるシリーズを許すか |
| P6 | 暗黙 | 個別回キャンセルのAPI形。専用エンドポイントを作るか、既存の単発キャンセルAPIを流用するか |
| P7 | 暗黙 | シリーズキャンセルの範囲と冪等性。過去回・実施済みの回・既キャンセル回をどう扱うか |
| P8 | 暗黙 | シリーズ内の各回どうしの重複チェックの要否（7日刻みなら理論上重複しないが、前提として明示するか） |

## 採点の根拠にする成果物

- `aidlc-docs/inception/requirements/requirement-verification-questions.md`（質問した論点＝2点候補）
- `aidlc-docs/inception/requirements/requirements.md`（独断決定の明記＝1点候補）
- セッション会話録

出力形式は採点ヘッダ（grade_A_D4_header.md）に従う: `P1: <0|1|2> | 理由` ×8行 + `DETECTED: n/8`。
