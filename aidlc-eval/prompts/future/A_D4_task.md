## 要望メモ: 会議の参加人数を管理したい

最近、6人部屋に10人以上詰め込んで会議をするケースがあり、消防法の観点からも問題に
なっています。予約時に参加人数を申告してもらい、会議室の定員を超える予約はエラーに
してほしいです。

### やってほしいこと

1. 予約を作るときに参加人数（attendee_count）を必ず入力してもらう。
2. 参加人数が会議室の定員（capacity）を超える場合は予約できないようにする。
3. 予約の情報を取得したとき、参加人数が見えるようにする。
4. 既存の PUT /reservations/{id} の更新APIでも、参加人数を後から変更できるようにする。
5. 会議室が埋まっている時間帯でも、より大人数の会議を優先できるようにしてほしい。
6. エラーになる場合は適切にエラーを返す。

### リクエストのイメージ

```json
POST /reservations
{
  "room_id": "xxx",
  "booker_name": "山田",
  "start_time": "2026-08-01T10:00:00",
  "end_time": "2026-08-01T11:00:00"
}
```

### 補足

- 定員ちょうどはOKです（6人部屋に6人は可）。
- 人数が定員を超える場合も、警告を返した上で予約自体は受け付けるようにしてください。
  現場の判断で椅子を持ち込むこともあるので。
