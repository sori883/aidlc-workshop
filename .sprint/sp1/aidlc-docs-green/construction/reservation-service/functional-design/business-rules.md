# 業務ルール (Business Rules) — reservation-service

確定した設計判断（ユーザー回答）:
- **BR-削除**: 会議室削除は active 予約があれば拒否（Q-F1=A）
- **BR-キャンセル冪等**: 再キャンセルは冪等成功（Q-F2=A）
- **BR-重複防止**: アプリ層で「チェック→挿入」を1トランザクション（Q-F3=A）
- **BR-過去日時**: 過去開始時刻の予約は拒否（Q-F4=B）

---

## 会議室 (Room)

- **BR-R1**: `name` は必須。空文字・空白のみは 400。
- **BR-R2**: `capacity` は 0 以上の整数。負数・非整数は 400。
- **BR-R3**: `equipment` は文字列のリスト。未指定は空リストとして扱う。
- **BR-R4**: `location` は任意。未指定は空文字。
- **BR-R5（更新）**: 更新時も BR-R1〜R4 を適用。存在しない `room_id` の更新は 404。
- **BR-R6（削除）**: `room_id` が存在しなければ 404。**その会議室に status=active の予約が1件でもあれば削除を拒否（409）**。active 予約が無ければ（キャンセル済みのみ含む）削除可。削除時、紐づく cancelled 予約の扱いは実装で確定（残置 or 併せて削除。既定: 残置しても参照整合上の実害はないが、Code Generation で単純化のため併せて物理削除も可）。

## 予約作成 (Reservation Create)

- **BR-C1（時刻順序）**: `start_time` < `end_time` であること。等しい/逆転は 400。
- **BR-C2（予約者）**: `booker_name` は必須、空不可（400）。`booker_email` は任意（形式の厳密検証は行わない）。
- **BR-C3（会議室存在）**: `room_id` が存在しなければ 404。
- **BR-C4（過去日時）**: `start_time` が**サーバー現在時刻より過去**の場合は 400。（境界: `start_time == now` は許可。判定は作成処理時点の現在時刻を基準とする）
- **BR-C5（重複防止）**: 対象会議室の status=active の既存予約と時間帯が重なる場合は 409。重なり判定は **BR-OV**。処理は「重複チェック→挿入」を単一トランザクション内で行い、競合下でも二重作成を防ぐ。

## 重なり判定 (Overlap) — BR-OV

- 2つの区間 `[s1, e1)` と `[s2, e2)` が重なるのは **`s1 < e2` かつ `s2 < e1`** のとき（半開区間）。
- **隣接は重ならない**: `e1 == s2`（例 10:00-11:00 と 11:00-12:00）は重複ではない → 予約可。
- **完全一致は重なる**: 同一 `[s, e)` は重複 → 409。
- **内包・部分重なりは重なる**: 10:00-11:00 に対し 10:30-10:45 / 10:30-11:30 / 09:30-10:30 はいずれも重複 → 409。
- **異なる会議室**は同一時間帯でも重複扱いしない → 予約可。
- **cancelled 予約は判定対象外**（その時間帯は再予約可能）。

## 予約キャンセル (Reservation Cancel)

- **BR-X1**: `reservation_id` が存在しなければ 404。
- **BR-X2**: status=active の予約はキャンセルし、status=cancelled に更新して 200。
- **BR-X3（冪等）**: 既に cancelled の予約への再キャンセルは、状態を変えず 200 で現在の予約を返す（冪等）。

## 予約取得・一覧 (Reservation Read)

- **BR-L1**: 一覧は既定で全予約（active/cancelled 両方）を返す。
- **BR-L2**: `room_id` 指定で該当会議室のみに絞り込み可能。
- **BR-L3**: 期間 `from` / `to` 指定で、その範囲に**重なる**予約に絞り込み可能（半開区間、BR-OV に準拠）。
- **BR-L4**: 存在しない `reservation_id` の個別取得は 404。

## 空き会議室検索 (Availability Search) — US-08

- **BR-A1**: `start` < `end` であること。逆転/等しいは 400。
- **BR-A2**: 指定 `[start, end)` に status=active の予約が**重ならない**会議室を返す（BR-OV 準拠、隣接は「空き」）。
- **BR-A3**: cancelled 予約はふさがりとみなさない。
- **BR-A4**: 該当なしは空リスト（200）。エラーにしない。
- **BR-A5（任意/将来）**: 収容人数での絞り込みは今回のコア対象外（余力があれば拡張）。

## HTTP ステータス方針（まとめ）
| 事象 | ステータス |
|---|---|
| 作成成功 | 201 |
| 取得/一覧/キャンセル成功 | 200 |
| 削除成功 | 204 |
| バリデーション違反（時刻順序・必須・過去日時・収容人数等） | 400 |
| 参照先が存在しない（room/reservation not found） | 404 |
| 重複予約 / active予約ありの会議室削除 | 409 |
